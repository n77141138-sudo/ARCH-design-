## 技能概述

Quote_Engine 是 Agent Ana 的核心計算技能，負責將廠商報價單（非結構化資料）轉化為符合公司毛利邏輯的正式對業主報價單。

---

## 核心能力

### 1. 數據解析（Data Parsing）

| 輸入格式 | 解析方式 | 狀態 |
| :--- | :--- | :---: |
| `.json` | 結構化直接讀取 | ✅ 已實裝 |
| `.xlsx / .xls` | Pandas 欄位映射 | ✅ 已實裝 |
| `.jpg / .png` | OCR 視覺辨識 | 🔜 規劃中 |
| `.pdf` | PDF 文字/影像擷取 | 🔜 規劃中 |

**解析欄位：**
- 項目名稱（品名）
- 規格 / 備註
- 數量
- 單位（坪、台、組、式、米…）
- 廠商單價（成本）

---

## 2. 利潤運算（Profit Calculation）

$$\text{客戶報價} = \text{廠商成本} \times (1 + \text{利潤率})$$

| 參數 | 預設值 | 調整範圍 | 設定位置 |
| :--- | :---: | :---: | :--- |
| 利潤率 | **20%** | 5% ～ 50% | `config.json` / Web UI 滑桿 |
| 加價倍率 | **1.20** | — | 自動計算 |

**範例：**
- 廠商成本 NT$10,000 × 1.20 = **客戶報價 NT$12,000**
- 廠商成本 NT$94,700（冷氣主機）× 1.20 = **NT$113,640**

**彈性設定：**
- 可依**工程類別**設定不同利潤率（於 JSON 的 `"markup"` 欄位個別指定）
- 未指定時，統一套用 `config.json` 的全域利潤率

---

## 3. 格式標準化（Output Standardization）

### 輸出格式一：Markdown 表格
統一輸出格式，可直接貼入 Excel 或文件系統。

```markdown
| # | 項目名稱 | 規格/備註 | 數量 | 單位 | 單價 | 總計 |
| :---: | :--- | :--- | :---: | :---: | ---: | ---: |
| 1 | 開關部份 | - | 26 | 切 | 1,080 | 28,080 |
| | **▶ 水電工程 小計** | | | | | **NT$ 119,760** |
```

### 輸出格式二：Excel（彙整審核表）
欄位：`#` / `工程類別` / `廠商` / `項目名稱` / `規格/備註` / `數量` / `單位` / `廠商單價(成本)` / `對客單價` / `廠商小計` / `對客小計` / `加價倍率`

### 輸出格式三：Excel（對業主正式報價單）
欄位：`#` / `項目名稱` / `規格/備註` / `數量` / `單位` / `單價` / `總計`

## 輸入規範（廠商報價 JSON 格式）

```json
[
  {
    "vendor": "廠商名稱",
    "category": "工程類別（空調/水電/木工…）",
    "items": [
      {
        "name": "項目名稱",
        "spec": "規格或備註（選填）",
        "qty": 1,
        "unit": "式",
        "unit_price": 10000,
        "markup": 1.2   // 選填，個別覆蓋全域利潤率
      }
    ]
  }
]
```

---

## 自動檢查機制

- ✅ 過濾 `unit_price = 0` 的無效項目
- ✅ 排除 Excel 暫存檔（`~$` 開頭）
- ✅ 依工程類別分組彙整，自動產出各類小計
- ✅ 加總金額交叉驗證（成本合計 ↔ 各類小計加總）
- ⚠️ 若品項模糊（單位空白、數量為 0），系統會在審核表標注，請人工確認

## 系統整合（Web UI）

```
使用者上傳廠商報價（JSON/Excel）
         ↓
POST /api/generate  { markup_rate, project_name, client_name }
         ↓
Quote_Engine 執行：解析 → 計算 → 驗證 → 格式化
         ↓
回傳：
  - markdown: string     (即時預覽)
  - stats: { cost, client_total, profit }
  - files: { review_excel, quote_excel, markdown }
```
